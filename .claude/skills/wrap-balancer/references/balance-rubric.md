# 한국어 텍스트 밸런싱 품질 루브릭 (wrap-balancer)

> 이 루브릭은 "wrap-balancer를 적용한 뒤 제목의 줄바꿈이 **실제로 더 나아졌는가**"를
> 전(before)·후(after) 렌더링을 근거로 판정하기 위한 기준이다. 한국어 조판의
> 특성(어절 단위 줄바꿈, `word-break: keep-all`)을 1급 기준으로 포함한다.
>
> 핵심 관점: 밸런싱은 **줄 수를 늘리지 않으면서**(높이 불변식) 각 줄의 너비를
> 고르게 만들고 마지막 줄의 외톨이(orphan)를 없애는 것이 목표다. 따라서 "전보다
> 유사한가"가 아니라 "전의 문제를 후가 **해소**했는가"를 본다. 줄바꿈이 눈에 띄게
> 바뀌는 것은 실패가 아니라 **기대된 동작**이다.

---

## 0. 입력

평가자는 다음을 근거로 판정한다(가능한 것을 모두 사용, 스크린샷은 필수):

- `before.png` — 밸런싱 미적용(브라우저 기본 greedy 줄바꿈) 렌더링 **[필수]**
- `after.png` — wrap-balancer 적용 렌더링 **[필수]**
- (선택) `metrics.json` — 줄 수·줄별 너비 등 객관 측정값. 있으면 **수치를 우선** 근거로 삼고
  스크린샷으로 교차 확인한다. 없으면 스크린샷만으로 판단한다.
- (선택) `compare.html` 스크린샷 — 전/후/네이티브를 나란히 본 참고용.

전·후는 **동일 텍스트·동일 너비·동일 폰트**에서 렌더링되었다고 가정한다(하니스가 보장).

---

## 1. 하드 게이트 (하나라도 위반하면 점수와 무관하게 `regression`)

밸런싱이 레이아웃을 **망가뜨리지 않았는지** 먼저 확인한다. 이 항목들은 점수가 아니라
통과/실패다.

| ID | 게이트 | 위반 = 실패 조건 |
|----|--------|------------------|
| **G1 · 줄 수 보존** | 후의 줄 수 ≤ 전의 줄 수 | 밸런싱이 줄을 **늘렸다**(같은 너비에서 후가 전보다 줄이 더 많다). 알고리즘 불변식(높이 유지) 위반 신호 — 보통 설정 오류. |
| **G2 · 넘침/잘림 없음** | 텍스트가 컨테이너 경계를 넘치거나(overflow) 잘리지(clip/ellipsis) 않음 | 후에서 글자가 카드/박스 밖으로 삐져나오거나 `…`로 잘림. |
| **G3 · 내용 보존** | 글자·단어가 누락/중복/변형되지 않음 | 후에서 텍스트 일부가 사라지거나 깨짐(예: 래핑 과정에서 노드 유실). |

> 게이트 통과는 "밸런싱이 해를 끼치지 않았다"는 최소 조건이다. 개선 여부는 §2에서 채점한다.

---

## 2. 채점 기준 (각 0–5점, 가중 합산 → 0–100)

점수 척도: **5 = 모범적 / 4 = 좋음 / 3 = 보통 / 2 = 미흡 / 1 = 나쁨 / 0 = 해당 축에서 후가 전보다 악화**.

| ID | 기준 | 가중치 | 무엇을 보는가 |
|----|------|:---:|------|
| **C1 · 고아 단어 제거** | 0.30 | 마지막 줄이 **한 단어/짧은 토막**으로 외롭게 남지 않는가. 전에 고아가 있었고 후에 해소되면 5점. 전에 없던 문제를 후가 만들면 0점. |
| **C2 · 줄 길이 균형** | 0.30 | 각 줄의 너비가 **고른가**. 가장 긴 줄과 짧은 줄의 차이가 전 대비 **줄었는가**. metrics.json이 있으면 줄너비 표준편차/(max−min) 감소로 확인. |
| **C3 · 한국어 어절 보존** | 0.20 | 줄바꿈이 **어절 경계(공백)** 에서 일어나는가. 단어(어절) 중간이 끊기지 않는가(`keep-all`). 단어 중간 끊김이 보이면 강한 감점. 긴 단일 토큰이 불가피하게 끊긴 경우는 예외(넘침 방지). |
| **C4 · 가독성·시각 안정감** | 0.12 | 전체적으로 더 **읽기 쉽고** 안정적인 덩어리로 보이는가. 줄 끝이 들쭉날쭉하지 않고 시선 흐름이 자연스러운가. |
| **C5 · 부작용 없음** | 0.08 | 정렬·자간·줄간격·인접 요소에 **악영향이 없는가**. 후가 전보다 나빠진 축이 하나도 없는가(회귀 없음). |

**총점** = Σ(점수 × 가중치) × 20 → 0–100 정수 반올림.
(가중치 합 = 1.0, 각 점수 최대 5이므로 5×1.0×20 = 100.)

---

## 3. 특수 케이스 (오판 방지)

밸런싱이 "안 바뀐 것"이 항상 실패는 아니다. 다음을 먼저 분기하라.

- **NO-OP · 한 줄 텍스트**: 전이 이미 **한 줄**이면(래핑 자체가 없음) 밸런싱은 변화 없음이 정상.
  → `verdict: "noop"`, `score: null`. "변화 없음"을 실패로 적지 말 것.
- **NO-OP · 이미 균형**: 전이 이미 여러 줄이지만 **충분히 균형**이고 고아도 없으면, 후 ≈ 전이 정상.
  → `verdict: "noop"`, 차이가 미미함을 명시. 점수를 매긴다면 높게(이미 좋음).
- **MARGINAL · 부분 개선**: 고아는 남았지만 균형은 좋아진 식의 **부분 개선**.
  → `verdict: "marginal"`, 무엇이 남았는지 `differences`에 적는다.
- **한국어인데 단어 중간이 끊긴다**: 거의 항상 `word-break: keep-all` 누락이 원인.
  → C3 감점 + `suggestions`에 "대상 요소에 `word-break: keep-all; overflow-wrap: anywhere;` 추가"를 반드시 적는다.

---

## 4. 판정(verdict) 결정 규칙

순서대로 적용한다(위가 우선).

1. 하드 게이트 위반 → `verdict: "regression"` (어떤 게이트인지 `differences`에 명시).
2. NO-OP 조건(한 줄 / 이미 균형) → `verdict: "noop"`.
3. 총점 **≥ 80** → `verdict: "improved"`.
4. 총점 **60–79** → `verdict: "marginal"`.
5. 총점 **< 60** → `verdict: "regression"` (개선 실패; 사실상 효과 없음 또는 악화).

> 기본 합격선은 **80점**이다. `improved` 또는 `noop`이면 적용을 그대로 둔다.
> `marginal`/`regression`이면 §5의 조치를 제안하고 재적용·재평가한다.

---

## 5. 점수가 낮을 때의 표준 조치 (suggestions로 출력)

- 단어 중간 끊김(C3↓) → `word-break: keep-all; overflow-wrap: anywhere;`를 대상 요소에 추가.
- 균형이 약함(C2↓) → `data-br-ratio`를 `1`(가장 압축)로. 기본이 1이 아니면 1로 올린다.
- 줄이 늘었다(G1) → 컨테이너 너비가 비정상적으로 좁지 않은지, 폰트/줄간격이 전·후 동일한지 확인.
  네이티브 경로와 JS 경로 결과가 다르면 `data-br-prefer-native="false"`로 JS 경로를 고정해 비교.
- 넘침(G2) → 컨테이너에 `min-width:0`/`overflow-wrap:anywhere`가 필요한지, 긴 URL/토큰이 있는지 확인.
- 변화 없음인데 변화를 기대했다 → 대상 셀렉터가 실제 제목 요소를 가리키는지, 스크립트가 로드·실행됐는지(`WrapBalancer` 전역 존재) 확인.

---

## 6. 출력 계약 (평가 에이전트가 반환할 JSON)

```json
{
  "verdict": "improved | marginal | noop | regression",
  "score": 0,
  "gates": { "G1_line_count": true, "G2_no_overflow": true, "G3_content_intact": true },
  "criteria": {
    "C1_orphan":      { "score": 0, "weight": 0.30, "note": "..." },
    "C2_evenness":    { "score": 0, "weight": 0.30, "note": "..." },
    "C3_word_break":  { "score": 0, "weight": 0.20, "note": "..." },
    "C4_readability": { "score": 0, "weight": 0.12, "note": "..." },
    "C5_no_side_fx":  { "score": 0, "weight": 0.08, "note": "..." }
  },
  "before_lines": 0,
  "after_lines": 0,
  "differences": ["전→후로 무엇이 바뀌었나(구체적으로)"],
  "suggestions": ["점수가 낮을 때 실행 가능한 다음 조치"],
  "reasoning": "1–3문장 한국어 요약"
}
```

규칙:
- `score`는 0–100 정수. `verdict:"noop"`이면 `score`는 `null` 가능.
- `gates` 중 하나라도 `false`면 `verdict`는 `regression`이어야 한다.
- `differences`/`suggestions`/`reasoning`은 **한국어**로 작성한다.
- 근거 없는 추측 금지. 스크린샷에서 직접 관찰되는 사실만 적는다. metrics.json이 있으면 수치를 인용한다.
